Wake actions

When a wake phrase is detected in a transcript, the configured command runs with the post-wake text in $transcript. Useful for routing captured intent to an agent (hermes-agent --input "$transcript"), creating reminders, or anything else a shell command can do.

Template variables: $transcript (post-wake text), $transcript_full (entire transcript), $conversation_id, $wake_phrase. All shell-escaped via shlex.quote, so quotes in the transcript can't break the command.
💡 Slow commands → use nohup to avoid timeouts
LLM agents and other long-running commands routinely take longer than the action's timeout. Wrap them so the command spawns into the background and omilog returns immediately:

nohup hermes-agent --input $transcript --conversation $conversation_id > /tmp/hermes.log 2>&1 &

The action logs exit 0 almost instantly; the real work continues on its own and writes to the log file. Trade-off: omilog never sees the agent's actual output. Drop nohup + & when you want the result captured in the invocation log.

Important: use the variables bare ($transcript), not wrapped in quotes ("$transcript"). Each variable is already shell-escaped; adding outer quotes breaks the escaping when the transcript contains apostrophes or double quotes.
